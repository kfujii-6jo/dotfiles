# PRレビュー

GitHub Pull Requestの差分をレビューし、フィードバックを提供します。

## 手順

1. PR番号または現在のブランチからPR情報を取得
2. `gh pr diff` でPRの差分を分析
3. 以下の観点でレビューを実施

### 1. 変更の概要
- PRの目的と変更の全体像を把握
- 影響範囲の特定

### 2. コード品質
- ロジックの正確性
- エラーハンドリング
- エッジケースの考慮
- コードの可読性

### 3. 設計・アーキテクチャ
- 既存コードとの一貫性
- 適切な抽象化
- 依存関係の妥当性

### 4. セキュリティ
- 入力バリデーション
- 認証・認可の確認
- 機密情報の取り扱い

### 5. テスト
- テストカバレッジ
- エッジケースのテスト
- テストの可読性

## 出力形式

```
## PR概要
[PRの目的と変更内容の要約]

## 総合評価
[Approve / Request Changes / Comment]

## 詳細レビュー

### 良い点
- [良い実装や設計]

### 改善提案
- [ファイル:行番号] [問題点と改善案]

### 質問・確認事項
- [確認したい点]
```

## コマンド

- PR情報取得: `gh pr view <number>`
- 差分取得: `gh pr diff <number>`
- 現在のブランチのPR: `gh pr view`
