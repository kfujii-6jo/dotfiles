# 簡易コードレビュー

一般的な問題に焦点を当てた迅速なコードレビューを実施します。

## 手順

提供されたコードまたはファイルを以下の観点でレビュー:

### 1. 正確性
- ロジックエラー
- オフバイワンエラー
- null/undefined の処理
- エッジケース

### 2. コード品質
- 命名の明確さ
- 関数の長さ（焦点が絞られているか）
- コードの重複
- マジックナンバー/文字列

### 3. 潜在的なバグ
- 競合状態
- メモリリーク
- 未処理の例外
- 型の不一致

### 4. ベストプラクティス
- エラーハンドリングパターン
- 入力バリデーション
- リソースのクリーンアップ
- APIの一貫性

## 出力形式

以下の構造でフィードバックを提供:

```
## 概要
[全体的な評価の要約]

## 発見された問題

### 重大
- [問題]: [場所] - [問題である理由]

### 提案
- [改善点]: [場所] - [メリット]

## 良い点
- [よくできている点]
```

## ガイドライン

- 具体的に: 行番号やコードスニペットを参照
- 建設的に: 問題だけでなく解決策を提案
- 優先順位をつける: 影響の大きい問題を先に
- 簡潔に: 簡易レビューは焦点を絞ったフィードバック
