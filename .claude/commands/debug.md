# デバッグ支援

体系的にバグを診断し解決を支援します。

## 手順

以下のデバッグ方法論に従う:

### 1. 問題を理解する
- 期待される動作は何か？
- 実際の動作は何か？
- いつから発生しているか？
- 再現可能か？

### 2. 情報を収集する
- エラーメッセージとスタックトレース
- 関連するログ
- 問題を引き起こす入力データ
- 環境の詳細

### 3. 仮説を立てる
- 可能性のある原因をリストアップ
- 可能性の高さでランク付け
- 最近の変更を考慮

### 4. 調査する
- コード実行パスを追跡
- 変数の状態を確認
- 動作が期待と乖離する箇所を特定
- 一般的なバグパターンを探す:
  - オフバイワンエラー
  - null/undefined 参照
  - 競合状態
  - 型変換の問題
  - 境界条件

### 5. 解決策を提案する
- 最小限の修正を提案
- 根本原因を説明
- 予防策を推奨

## 出力形式

```
## 問題の概要
[問題の明確な説明]

## 分析

### エラーの詳細
[エラーメッセージ、スタックトレース]

### 調査
[実施した手順、発見事項]

### 根本原因
[実際に問題を引き起こしているもの]

## 解決策

### 修正
[必要なコード変更]

### 説明
[なぜこれで問題が解決するか]

### 予防
[同様の問題を防ぐ方法]

## テスト
[修正が機能することを確認する方法]
```

## ガイドライン

- 最も単純な説明から始める
- 仮定せず、証拠で検証する
- ロジックを疑う前に入力を確認
- より広いコンテキストを考慮
- リグレッション防止のためのテスト追加を提案
