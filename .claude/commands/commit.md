# コミットメッセージ生成

ステージされた変更に基づいて、Conventional Commits形式のコミットメッセージを生成します。

## 手順

1. `git diff --cached` でステージされた変更を分析
2. 変更の種類を特定:
   - `feat`: 新機能
   - `fix`: バグ修正
   - `docs`: ドキュメントのみ
   - `style`: フォーマット変更（コード変更なし）
   - `refactor`: 動作変更を伴わないコード再構成
   - `perf`: パフォーマンス改善
   - `test`: テストの追加・更新
   - `chore`: ビルド、設定、ツール関連の変更
3. スコープを決定（任意）: 影響を受けるモジュール、コンポーネント、領域
4. 簡潔な件名を記述（最大50文字、命令形）
5. 必要に応じて本文を追加（72文字で折り返し）
6. 該当する場合はIssueを参照

## 出力形式

```
<type>(<scope>): <subject>

<body>

<footer>
```

## ルール

- 件名: 命令形、ピリオドなし、最大50文字
- 本文: 「何を」「なぜ」を説明（「どのように」ではない）
- フッター: Issueを参照（例: `Closes #123`）
- 現在形を使用（「add feature」であり「added feature」ではない）

## 例

```
feat(auth): OAuth2ログイン機能を追加

GoogleとGitHubプロバイダーを使用したOAuth2認証フローを実装。
これにより、ユーザーは新規アカウントを作成せずにサインインできます。

Closes #42
```
